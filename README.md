# 🍽️ 智能就餐管理系统

## 📌 作品信息

| 项目 | 内容 |
|------|------|
| **作品名称** | 智能就餐管理系统 |
| **参赛赛道** | 学习工作 / 社会服务 |
| **开发工具** | TRAE IDE (SOLO模式) |
| **作品类型** | Web应用（纯前端HTML + 可部署服务端版） |
| **适用场景** | 企业/学校/机关单位食堂就餐管理 |


## 🎯 项目背景

食堂/餐厅传统管理方式普遍存在以下痛点：
- 人员出差、请假等情况变化频繁，人工统计就餐人数困难，备餐不准导致食材浪费或供不应求
- 就餐时间集中，多窗口排队选菜，人流压力大、场面混乱
- 菜品金额需人工核算，耗时长且容易出错
- 就餐数据无法有效留存分析，管理决策缺乏数据支撑

**本项目正是为解决以上问题而生。** 通过AI辅助开发，实现从订餐到结算的全流程数字化管理。


## 🛠️ 技术实现

本项目完全使用 **TRAE IDE (SOLO模式)** 通过自然语言对话完成开发，无需手写大量代码，体现了AI辅助编程“让有想法的人先把东西做出来”的核心理念。

| 模块 | 技术方案 |
|------|----------|
| 前端 | 纯HTML + CSS + JavaScript（单页面应用） |
| 数据存储 | localStorage（本地存储，无需数据库） |
| 部署方式 | 双击HTML直接运行 / 部署到任意Web服务器 |

**作品包含文件：**
1. `index.html` — 完整前端应用，可直接双击运行
2. `server版压缩包` — 优化后的完整项目包，可直接部署到服务器


## 🎨 核心功能

**1. 智能订餐模块**
员工可在线查看当日菜谱，按日期和餐别选择菜品，订单自助提交，金额自动核算。

**2. 人员管理模块**
支持员工名单的导入、管理和批量操作，角色权限划分清晰。

**3. 就餐统计模块**
实时统计就餐人数，菜品热度分析辅助精准备餐，就餐数据留档支持历史查询。

**4. 用户认证模块**
多角色登录，不同角色不同操作权限，登录状态自动保持。

## 📖 使用说明

**方式一（推荐演示）：** 双击打开 `index.html`，使用测试账号登录即可体验。

**方式二（生产部署）：** 解压server版压缩包，上传至Web服务器根目录，通过域名或IP访问。

**核心流程：** 登录 → 查看菜单 → 在线订餐 → 订单提交 → 管理员统计备餐
